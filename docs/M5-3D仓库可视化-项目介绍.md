# M5: 3D 仓库可视化 - 项目介绍

> 更新时间：2026-06-26  
> 模块状态：✅ 已实现（Demo 级）

## 模块概览

**M5: 3D 仓库可视化** 在 M3 仓储数据之上，用 **Three.js** 呈现库位占用与风险分布，并在履约拣货场景下展示 **推荐拣货路径**，帮助仓库人员直观理解「去哪拣、先拣哪」。

与真实 WMS 的区别：路径排序为演示级规则（按 X/Z 坐标），非最优 TSP 算法。

---

## 核心能力

| 能力 | 说明 |
|------|------|
| 库位风险热力 | empty / low / normal / full 四档，颜色映射到 3D 方块 |
| 仓库切换 | 多仓库下拉，联动布局与风险数据 |
| 拣货导航 | 从履约中心带入 `shipment_id`，展示路径点顺序与 HUD |
| 降级表格 | 无 WebGL 或移动端自动切换为库位列表视图 |
| 侧栏筛选 | 按库位编码搜索、风险图例开关、「仅看有货」 |

## 数据流

```text
┌──────────────┐     GET layout / risk_map      ┌─────────────────┐
│ warehouse-3d │ ─────────────────────────────▶│ warehouse.js    │
│   (Vue)      │     GET picking_route          │ shipment.js     │
└──────┬───────┘ ─────────────────────────────▶└────────┬────────┘
       │                                                  │
       ▼                                                  ▼
 Three.js 渲染库位方块                          warehouse_locations
 + 路径连线 + 图例着色                          stocks / shipment_items
```

### 布局 API（`GET /api/proj/warehouse/layout`）

返回仓库、库区、货架、库位及 **pos_x / pos_y / pos_z / capacity**，供前端构建 3D 网格。

### 风险地图（`GET /api/proj/warehouse/risk_map`）

按库位当前数量与容量计算占用率，映射为：

| 等级 | 条件 | 颜色（示意） |
|------|------|--------------|
| empty | qty ≤ 0 | 灰色 |
| low | 0 < 占用率 ≤ 15% | 橙色 |
| normal | 15% ~ 90% | 绿色 |
| full | ≥ 90% | 红色 |

### 拣货路径（`GET /api/proj/shipment/picking_route`）

入参 `shipment_id`，返回有序 `points[]`（库位坐标、SKU、数量、seq）。排序策略：**pos_x 优先，pos_z 次之**。

## 使用路径

### 1. 仅看库位态势

1. 顶部模块进入 **3D 仓库**（或仓储相关菜单）
2. 选择仓库 → 旋转/缩放场景
3. 点击库位查看详情卡片；侧栏可搜索库位编码

### 2. 拣货导航（与 M4 联动）

1. 履约中心对发货单 **开始拣货**
2. 进入 3D 页时 URL 携带 `shipment_id`（或从履约页跳转）
3. 顶栏显示发货单号与路径点数；场景中高亮路径与当前步骤

### 3. AI 业务助手

问「3D 仓库怎么用？」「仓库今天怎么拣货？」会命中 playbook `warehouse-picking`，返回分步链接。

## 权限与菜单

- 顶层菜单 **「仓储管理」** 使用 `permissionCode: 'menu:fulfillment'`（与履约共用，无独立 `menu:warehouse`）
- 路径：`/view/dashboard/warehouse-3d?proj_key=retail&warehouse_id=...`
