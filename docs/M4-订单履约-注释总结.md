## 智能分仓算法详解

### 算法目标
选择能完全满足订单所有 SKU 需求且库存分布最均衡的仓库

### 算法步骤

```javascript
function findWarehouseForItems(trx, tenantId, items) {
  // 1. 批量拉取 active 仓库 + stocks 矩阵（避免 N+1）
  // 2. 对每个仓库：检查各 SKU available_qty 是否满足需求
  // 3. 在可履约仓中选 min(available) 最大的仓
  return { warehouse, warehouseId }
}
```

### 为什么选择"最小可用库存最大"

**示例场景**：订单需要 SKU1(10件) + SKU2(10件)

| 仓库 | SKU1 可用 | SKU2 可用 | minAvailable | 是否选择 |
|------|-----------|-----------|--------------|----------|
| 仓库 A | 100 | 15 | 15 | ❌ |
| 仓库 B | 50 | 50 | 50 | ✅ 最优 |
| 仓库 C | 20 | 80 | 20 | ❌ |

**选择仓库 B 的原因**：
1. **库存分布均衡**：两个 SKU 库存都充足
2. **降低缺货风险**：不会因某个 SKU 快售罄而影响后续订单
3. **减少补货压力**：库存消耗更均衡

**对比其他策略**：
- ❌ **选总库存最大**：可能某个 SKU 过多，另一个快售罄
- ❌ **选平均库存最大**：忽略了最薄弱的 SKU
- ✅ **选最小库存最大**：关注"木桶短板"，最保险

## 拣货路径规划

### 路径规划策略

```javascript
// 1. 获取发货单所有明细的库位坐标
const points = []
for (const item of items) {
  const loc = await db('warehouse_locations')
    .where({ location_id: item.suggested_location_id })
    .first()
  
  points.push({
    location_code: loc.location_code,
    sku_code: item.sku_code,
    qty: item.qty,
    pos_x: loc.pos_x,
    pos_y: loc.pos_y,
    pos_z: loc.pos_z,
  })
}

// 2. 按 (pos_x, pos_z) 排序（X 轴优先，Z 轴次要）
points.sort((a, b) => {
  if (a.pos_x !== b.pos_x) return a.pos_x - b.pos_x
  return a.pos_z - b.pos_z
})

// 3. 标记拣货顺序
points.forEach((p, i) => { p.seq = i + 1 })
```

### 路径数据结构

```javascript
{
  shipmentId: 'ship_123',
  warehouseId: 'wh_456',
  shipmentNo: 'SHP20260622123456ABC',
  points: [
    {
      seq: 1,                        // 拣货顺序
      location_id: 'loc_1',
      location_code: 'A-01-01',
      sku_id: 'sku_1',
      sku_code: 'SKU001',
      qty: 10,
      pos_x: 0,                      // X 坐标
      pos_y: 0,                      // Y 坐标（高度）
      pos_z: 0,                      // Z 坐标
    },
    {
      seq: 2,
      location_code: 'A-01-02',
      // ...
      pos_x: 0,
      pos_z: 1,
    },
    {
      seq: 3,
      location_code: 'A-02-01',
      pos_x: 1,
      pos_z: 0,
    }
  ]
}
```

### 3D 路径可视化

```javascript
// Three.js 渲染拣货路径
const { points } = await api.get('/api/proj/shipment/picking_route')

// 1. 绘制路径点
points.forEach((point, i) => {
  // 创建标记
  const marker = createMarker(point.seq)
  marker.position.set(point.pos_x, point.pos_y, point.pos_z)
  scene.add(marker)
  
  // 绘制路径线
  if (i > 0) {
    const prev = points[i - 1]
    const line = createLine(
      new THREE.Vector3(prev.pos_x, prev.pos_y, prev.pos_z),
      new THREE.Vector3(point.pos_x, point.pos_y, point.pos_z)
    )
    scene.add(line)
  }
})
```

## Excel 导入订单

### Excel 格式要求

| 列名（中/英） | 说明 | 必填 | 示例 |
|------|------|------|------|
| 店铺名称 / store_name | 须为租户内已启用店铺 | ✅ | Demo 旗舰店 |
| SKU编码 / sku_code | 有效在售 SKU | ✅ | SKU-DEMO-001 |
| 数量 / qty | 购买数量 | ✅ | 10 |
| 单价 / unit_price | 商品单价 | 建议填 | 99.00 |
| 订单号 / order_no | 同号多行合并为一单 | 可选 | ORD20260622001 |
| 客户名称 / customer_name | 自动建档客户 | 可选 | 张三 |

不支持收货地址/电话列；客户信息仅通过「客户名称」关联。

### 导入规则

1. **订单合并**
   - 同 `order_no` 的多行合并为一个订单的多条明细
   - 未填订单号时，每行独立生成新订单号

2. **数据验证**
   - 店铺名称须在 `stores` 中存在且 `status=active`
   - SKU 编码须存在且 `status=active`
   - 数量必须 > 0
   - 租户内订单号不可重复

3. **同步处理**
   - `POST /api/proj/order/import` **同请求内**完成解析与写入
   - 立即返回 `{ batchId, total, success, fail, errors }`
   - 批次状态写入 `order_import_batches.status`：`processing` → **`done`**
   - 可选：`GET /api/proj/order/import_result?batch_id=...` 查询批次与错误明细

### 导入结果示例

```javascript
{
  batchId: 'ibatch_xxx',
  total: 100,
  success: 95,
  fail: 5,
  errors: [{ rowNo: 10, reason: 'SKU 不存在: SKU999', raw: {} }]
}
```

## 设计亮点

### 1. 智能分仓算法
基于"最小可用库存最大"策略，选择库存分布最均衡的仓库：

```javascript
// 关注"木桶短板"
minAvailable = Math.min(...skus.map(s => s.available_qty))
best = warehouses.maxBy(wh => wh.minAvailable)
```

### 2. 拣货路径规划
按 3D 坐标排序，减少拣货移动距离：

```javascript
// X 轴优先，Z 轴次要
points.sort((a, b) => {
  if (a.pos_x !== b.pos_x) return a.pos_x - b.pos_x
  return a.pos_z - b.pos_z
})
```

### 3. 库存锁定机制
支付时锁定库存，出库时消耗：

```javascript
// 支付时
lock_id = await stock.lock({ sku_id, qty })
await order_items.update({ lock_id })

// 出库时
await stock.outbound({ sku_id, qty, lock_id })
```

### 4. Excel 同步导入

上传后在同一 HTTP 请求内完成解析与落库，返回 `batchId`；大批量场景未做异步队列（Demo 够用）。

```javascript
// POST /api/proj/order/import → { batchId, total, success, fail, errors }
// GET /api/proj/order/import_result?batch_id=...
```
