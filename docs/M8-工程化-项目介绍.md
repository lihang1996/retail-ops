# M8: 工程化 - 项目介绍

> 更新时间：2026-06-26  
> 模块状态：✅ 已实现（Docker / CI / 脚本 / 单测）

## 模块概览

**M8: 工程化** 保障 retail-ops 可 **本地开发、构建部署、数据初始化、自动化校验**，并与同级 **elpis** 组成 monorepo（`dom/elpis` + `dom/retail-ops`）。

---

## npm 脚本（package.json）

| 命令 | 用途 |
|------|------|
| `npm run dev` | 本地开发（nodemon） |
| `npm run build:local` | local 环境前端构建 |
| `npm run build:prod` | 生产前端构建 |
| `npm run start:prod` | `NODE_ENV=production` 启动 |
| `npm run migrate` | Knex migrations |
| `npm run seed` | Knex seeds |
| `npm run seed:demo` | migrate + 全量 demo seed |
| `npm run reset:demo` | 清理业务数据后重跑 seed |
| `npm run lint` | 静态检查（lint-lite） |
| `npm run test:unit` | 单元测试（tests/unit/*.test.js） |
| `npm run check:routes` | 路由 / permission-map / router-schema 一致性 |
| `npm run check:dist` | 构建产物跟踪策略 |
| `npm run e2e:smoke` | API 冒烟（需服务运行） |
| `npm run e2e:golden` | 黄金链路脚本 |
| `npm run test:concurrency` | 订单并发 smoke（**未**默认进 CI） |
| `npm run test:pagination` | 分页 smoke（CI e2e job 使用） |

## CI（`.github/workflows/ci.yml`）

### build job

1. checkout
2. 安装 `../elpis`（若存在）+ retail-ops
3. `npm run lint`
4. `npm run test:unit`
5. `npm run check:routes`
6. `npm run build:prod`

### e2e job（needs: build）

1. MySQL 8 service 容器
2. `SEED_ON_START=true`
3. `node scripts/init-db.js`
4. 启动 `serve.js`
5. `npm run e2e:smoke`
6. `npm run test:pagination`

若无法推送 workflow，可参考 `deploy/ci.yml.example`（内容较旧，以仓库内 `ci.yml` 为准）。

## 本地开发

```bash
cd ../elpis && npm install
cd ../retail-ops && npm install
npm run migrate && npm run seed
npm run build:prod && npm run dev
```

访问：http://localhost:8090/view/login

MySQL：本地 `config/config.local.js` 默认 **3306**；Docker Compose 宿主机映射见 `.env.example` 的 `DB_PUBLISH_PORT`（默认 **3307**）。

---

## Docker 部署

```bash
cp .env.example .env
docker compose up -d --build
```

- 登录：http://localhost:8090/view/login
- 健康：`GET /health`；`GET /health/detail`（含 DB/Redis 探测）

### init-db 与 seed

`scripts/init-db.js`：

- 始终执行 **migrate**
- **seed** 条件：`SEED_ON_START=true` **或** `tenants` 表为空
- 已有数据的容器默认 skip seed（避免覆盖）

## 路由与权限校验

新增 API 须同步：

1. `app/router/*.js`
2. `app/router-schema/*.js`
3. `app/common/permission-map.js`

运行 `npm run check:routes` 验证。

**elpis 框架路由**（不在 `app/router/`）：`GET /api/project/list`、`GET /api/project` — 由 permission-map 声明，供项目列表页使用。

**健康检查**：`GET /health`、`GET /health/detail` — 免 JWT（见 route-manifest 豁免规则）。
